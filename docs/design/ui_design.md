# UIデザイン案（簡素で見やすい前提）

## 1. デザイン方針
- シンプル・少色・高コントラストで可読性を最優先。
- 情報の縦流れを意識し、操作を左→右（アップロード→ビュー→操作）で配置。
- 余白を多めに取り、重要操作は一列にまとめる。
- 1カラム基調、横幅がある場合は2カラム（左:ビュー/右:操作パネル）に自動切替。

## 2. レイアウト（デスクトップ）
```
---------------------------------------------------------
 Header: タイトル | 簡易ヘルプリンク
---------------------------------------------------------
 Main:
  [UploadCard]  (未読み込み時センター表示。読み込み後は左上に縮退)
  [ViewerArea]  PDFキャンバス。下にページコントロール。
  [SidePanel]   操作まとめ:
    - PageControls (Prev / Page input / Next)
    - RotateControls (↺ / ↻ / OCR / 提案適用)
    - ZoomControls (+ / - / Fit)
    - SaveCard (適用/ダウンロード)
  Toast/Loading overlays
---------------------------------------------------------
 Footer: ショートカット案内・バージョン
```

## 3. レイアウト（モバイル）
- 1カラム縦積み。Viewer上、Controls下。
- 上部に Upload ボタン、Viewer 直下に PageControls、RotateControls、ZoomControls、Save ボタン。
- 固定ヘッダは避け、上部にタイトル + ヘルプの小さなバーを置く。

## 4. カラーパレットとタイポ
- 背景: `#f7f7f8`
- カード: `#ffffff`
- プライマリ: `#2563eb`（ボタン/リンク）、ホバー `#1d4ed8`
- セカンダリ枠線: `#e5e7eb`
- テキスト: `#0f172a`、サブ `#475569`
- フォント: システムUI系（Segoe UI / sans-serif）で軽量

## 5. コンポーネント仕様
- Header
  - 左: プロダクト名（例: PDF Rotator）
  - 右: ショートカット一覧リンク、簡易ヘルプ（モーダル）
- UploadCard
  - ドラッグ&ドロップ + ボタン
  - サブテキスト: 拡張子/サイズ上限、プライバシー説明（ローカル処理）
  - エラー時: 枠線を赤、メッセージを表示
- ViewerArea
  - キャンバス中央配置、背景チェックパターンなしで白ベース
  - ローディング: スピナーと「読込中…」の短文
  - エラー: カード内にメッセージ
- PageControls
  - Prevボタン、ページ番号入力（`x / total`）、Nextボタン
  - 入力は数値のみ、バリデーションエラーは赤枠
- RotateControls
  - 左右回転ボタン（↺ -90, ↻ +90）
  - OCRボタン（「向き推定」）: 実行中はスピナー
  - 提案表示: `推定: 90° (信頼度 0.82)`（現在ページ以降へ自動適用）
- ZoomControls
  - `-` `100%` `+` と「幅に合わせる」トグル
- SaveCard
  - プライマリボタン「適用して保存」
  - サブリンク「元PDFを再選択」
- Toast/Feedback
  - 右下（デスクトップ）/ 下部（モバイル）に短いフィードバック
  - 色: 成功 `#16a34a`、警告 `#d97706`、エラー `#dc2626`

## 6. 状態に応じた表示
- 初回未ロード: UploadCard を中央、その他は非表示。
- ロード中: ViewerArea にローディング。Controlsは disabled。
- 正常: Viewer + Controls 有効。
- OCR無効/失敗: OCRボタンを disabled/再試行表示、提案部をグレーアウト。
- 保存中: Saveボタンにスピナー、重複クリック防止。

## 7. ショートカット案内（UI内）
- ヘッダのヘルプリンクかフッタに簡易表を常設:
  - →: 右回転
  - ←: 左回転
  - ↓/↑: ページ移動
  - Ctrl+S: 適用して保存

## 8. アクセシビリティ
- 主要ボタンに `aria-label`、フォーカスリングを太めに表示（色: プライマリ）。
- コントラストは WCAG AA 以上を確保（背景と文字比 > 4.5:1）。
- キーボード操作: タブ移動で全ボタン/入力に到達可能。

## 9. レイアウトブレークポイント例
- `>= 1024px`: ViewerとSidePanelの2カラム（比率 2:1）。SidePanelはスクロール固定。
- `< 1024px`: 1カラム縦積み。サイドパネル要素は Viewer 下に順番に配置。

## 10. 実装上のスタイル指針
- CSS: シンプルなカスタムプロパティを使用。
  - `--bg: #f7f7f8; --card: #fff; --text: #0f172a; --muted: #475569; --border: #e5e7eb; --primary: #2563eb; --primary-hover: #1d4ed8;`
- 影は最小限（カード: `0 1px 4px rgba(0,0,0,0.06)`）。
- 角丸は小さめ（6px）。ボタンはミディアムサイズ（高さ40px前後）。

## 11. 既知の簡素化
- サムネイル一覧やページリストは持たず、ページ番号入力とプリフェッチで対応。
- ツールバーを1段にまとめ、階層を深くしない。
