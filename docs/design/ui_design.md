# UIデザイン案（Editorial Design System）

最終更新: 2025-12-22

## 1. デザイン方針
- **Editorial/Magazine スタイル**: 雑誌的な洗練されたレイアウトを目指す。
- ミニマルで高コントラスト、ボーダーとラインで区切りを表現。
- 情報の縦流れを意識し、操作を左→右（アップロード→ビュー→操作）で配置。
- 余白を多めに取り、重要操作は一列にまとめる。
- 1カラム基調、横幅がある場合は2カラム（左:ビュー/右:操作パネル）に自動切替。

## 2. レイアウト（デスクトップ）
```
---------------------------------------------------------
 Header: タイトル | 簡易ヘルプリンク
---------------------------------------------------------
 Main:
  [UploadCard]  (未読み込み時センター表示。読み込み後は左上に縮退)
  [ViewerArea]  サムネイル一覧（4-5列、スクロール）。選択数と状態表示。
  [SidePanel]   操作まとめ:
    - OCRControls (向き推定/しきい値/状態)
    - RotateControls (選択を±90°)
    - SaveCard (適用/ダウンロード、選択解除)
  Toast/Loading overlays
---------------------------------------------------------
 Footer: ショートカット案内・バージョン
```

## 3. レイアウト（モバイル）
- 1カラム縦積み。Viewer上、Controls下。
- 上部に Upload ボタン、Viewer 直下に OCR/Rotate/Save 操作を縦に配置。
- 固定ヘッダは避け、上部にタイトル + ヘルプの小さなバーを置く。

## 4. カラーパレットとタイポグラフィ（Editorial Design System）
- 背景: ダーク系（`#0a0a0a`）、ノイズテクスチャオーバーレイ
- サーフェス: `#161616`（カード）/ `#121212`（elevated）
- アクセント: 朱色（`#c7442f`）、ホバー時 `#d85540`
- テキスト:
  - Primary: `#f7f5f0`
  - Secondary: `#a8a8a8`
  - Tertiary: `#666666`
  - Display: `#ffffff`
- ボーダー: 薄い白（6%/12%透明度）
- フォント:
  - Display: Cormorant Garamond（イタリック見出し）
  - Body: DM Sans + Noto Sans JP
  - Mono: JetBrains Mono（ラベル/バッジ）

## 5. コンポーネント仕様
- Header
  - 左: プロダクト名（例: PDF Rotator）
  - 右: ショートカット一覧リンク、簡易ヘルプ（モーダル）
- UploadCard
  - ドラッグ&ドロップ + ボタン
  - サブテキスト: 拡張子/サイズ上限、プライバシー説明（ローカル処理）
  - エラー時: 枠線を赤、メッセージを表示
- ViewerArea
- 4-5列のサムネイル一覧をスクロール表示（大量ページは仮想スクロール）
  - クリック/ドラッグで複数選択、選択カードを強調
  - ダブルクリックでフロートモーダルの拡大プレビュー
  - ローディング: サムネイル生成中のピル表示
  - エラー: トーストで通知
- OCRControls
  - しきい値入力と向き推定ボタン
  - 推定ステータス/進捗/信頼度表示（現在ページ以降へ自動適用）
- RotateControls
  - 選択ページの回転ボタン（-90 / +90）
- SaveCard
  - プライマリボタン「適用して保存」
  - サブアクション「選択解除」
- Toast/Feedback
  - 右下（デスクトップ）/ 下部（モバイル）に短いフィードバック
  - 色: 成功 `#16a34a`、警告 `#d97706`、エラー `#dc2626`

## 6. 状態に応じた表示
- 初回未ロード: UploadCard を中央、その他は非表示。
- ロード中: ViewerArea にローディング。Controlsは disabled。
- 正常: Viewer + Controls 有効。
- OCR無効/失敗: OCRボタンを disabled/再試行表示、提案部をグレーアウト。
- 保存中: Saveボタンにスピナー、重複クリック防止。

## 7. ショートカット案内（UI内）
- ヘッダのヘルプリンクかフッタに簡易表を常設:
  - Ctrl/Cmd + →: 選択を +90° 回転
  - Ctrl/Cmd + ←: 選択を -90° 回転
  - Ctrl/Cmd + ↑/↓: 選択を 180° 回転
  - Esc: 選択解除（プレビュー表示中は閉じる）
  - Ctrl/Cmd + S: 適用して保存

## 8. アクセシビリティ
- 主要ボタンに `aria-label`、フォーカスリングを太めに表示（色: プライマリ）。モーダル内はフォーカストラップ。
- コントラストは WCAG AA 以上を確保（背景と文字比 > 4.5:1）。
- キーボード操作: タブ移動で全ボタン/入力に到達可能。

## 9. レイアウトブレークポイント例
- `>= 1024px`: ViewerとSidePanelの2カラム（比率 2:1）。SidePanelはスクロール固定。
- `< 1024px`: 1カラム縦積み。サイドパネル要素は Viewer 下に順番に配置。

## 10. 実装上のスタイル指針
- CSS: カスタムプロパティによるEditorial Design System。
  ```css
  --bg: #0a0a0a;
  --surface: #161616;
  --accent: #c7442f;
  --text-primary: #f7f5f0;
  --border: rgba(255, 255, 255, 0.06);
  --shadow-canvas: 0 24px 60px rgba(0, 0, 0, 0.7);
  ```
- 影: 控えめ〜中程度（`shadow-sm` / `shadow-md` / `shadow-lg`）。
- ボーダーとラインで区切りを表現し、グロー効果は使用しない。
- フォント: 見出しはセリフ系イタリック、本文はサンセリフ、ラベルはモノスペース。

## 11. 既知の簡素化
- サムネイル一覧で一覧性を優先し、ページ送りUIは省略。
- ツールバーを1段にまとめ、階層を深くしない。
