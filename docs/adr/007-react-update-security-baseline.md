# ADR-007: React最新版更新とセキュリティ監査の基準化

## ステータス
採用済み（2025-12-23）

## 背景
- フロントエンド基盤の安定性と保守性を維持するため、Reactの最新版追従が必要。
- 依存関係のセキュリティ状態を継続的に確認し、重大な脆弱性を早期に検知したい。

## 決定
- React/React DOMは最新パッチ版へ更新する。
- 更新のたびに `npm audit`（frontend/server）を実行し、結果を記録する。
- 依存更新は最小影響を優先し、まずパッチ更新から適用する。

## 根拠
- パッチ更新は後方互換性が高く、既存機能への影響を抑えられる。
- `npm audit` の定期確認は既知の脆弱性を早期に把握できる。

## トレードオフ
- 依存更新頻度が上がり、検証コストが増える。
- 監査結果がゼロでも記録が増える。

## 影響範囲
- フロントエンド依存（React/React DOM/型定義）の更新。
- セキュリティ監査の実行手順（CI/手動）。
